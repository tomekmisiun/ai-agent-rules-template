# Testing Rules

Adapt to the target repository's test framework and layout.

## Integrity

- New or changed behavior MUST have tests unless explicitly docs-only.
- Bug fixes SHOULD include regression tests.
- MUST NOT delete or skip tests without user approval.

## Layout

- Tests live in the directory structure defined by the target repository.
- Name tests consistently with project conventions.

## Coverage

- Follow the coverage policy defined by the target repository if one exists.
- Do not lower coverage thresholds without explicit approval.

## Commands

Use commands from `.ai-rules/validation.md` and the target README.
